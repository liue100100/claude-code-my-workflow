# Session log — 2026-07-06 — F2 vs AEMO QED direction-cost reconciliation

## Goal
User flagged that AEMO's QED Q2-2024 SA direction-cost chart (`Direction/AEMO_direction_cost.xlsx`, Figure 85) does not match our F2 proposal figure. Diagnose, then fix F2.

## Diagnosis
1. **Different cost concept.** AEMO plots the NER 3.15.8 cost-recovery amount (`cra` column in our parsed reports) = compensation_payment + additional_compensation + ie_fee − retained_trading_amount. F2 plotted `compensation_payment` only. Proof: summing `cra` by start quarter reproduces AEMO's Q2-23 value to machine precision (23.058985353704916) and Q4-22 within $7k — AEMO's spreadsheet numbers are literally sums of the CRA column from the same compensation reports.
2. **Coverage hole.** F2's `BOUNDARY <- "202310"` dropped cost-report rows in Oct 2023 while the per-DUID events file only begins 2023-10-27 → Oct 1–27, 2023 (~$7.1m CRA) missing entirely. Explains Q4-23 (ours 12.4 vs AEMO 27.4).
3. **Quarter assignment.** AEMO groups whole events by quarter; multi-week rows assigned to start quarter shuffle $2–4m between neighbours. Residual gaps (worst Q3-22 −$4.5m) are AEMO's convention + estimates, not data errors.
4. The committed F2 CSV also predated the +10h timestamp fix (stale on top of everything).

## Fix (proposal_figures.R, F2 block)
- Line now plots CRA (fill-in formula when `cra` is NA).
- Source split at the actual coverage seam (`max(direction_end)` = 2023-10-27 16:30), not the month label.

Reproduces AEMO's QED "percentage of time of directions" series within 0.1pp in 6 of 9 quarters (Q3-22 and Q3-23 to 4 decimals; worst Q4-22 −2.3pp) — strong external validation of the parsed events data and the timestamp fix.
